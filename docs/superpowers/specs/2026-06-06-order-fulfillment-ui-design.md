# 订单履约页 UI 设计记录

日期：2026-06-06

## 目标

将订单履约页从摘要卡片 + 卡片列表，收敛为更适合桌面运营场景的任务型表格工作台，不改变现有业务逻辑、IPC、数据库或接口行为。

## 设计结论

1. 首屏采用任务优先布局。
2. 顶部统计从大卡片改为紧凑 KPI 条。
3. 保留任务提醒优先级，弱化纯业务总览。
4. 筛选区拆分为基础筛选常驻，高级筛选折叠展开。
5. 主体改为单表格主视图，不再强调卡片视图切换。

## 页面结构

1. `PageHeader`
2. `今日任务 KPI 条`
3. `基础筛选 + 高级筛选折叠区`
4. `批量操作条`
5. `订单表格`

## KPI 条

主 KPI：

- 待发货
- 待归还
- 逾期异常

次级 KPI：

- 全部订单
- 租用中
- 已结束

所有 KPI 均支持点击后切换当前筛选视图。

## 筛选策略

基础筛选：

- 关键词
- 店铺
- 主状态

高级筛选：

- 型号编码
- 支付状态
- 免押状态
- 开始日期
- 结束日期

## 表格列

- 订单信息
- 客户
- 设备
- 租期
- 状态
- 金额
- 操作

## 状态列规则

状态列并排展示三个 Badge：

- 支付状态
- 履约状态
- 免押状态

其中履约状态优先表达待发货、待归还、逾期异常、租用中、已结束。

## 行操作规则

主操作根据订单阶段动态变化：

- 待发货：`去发货`
- 租用中：`续租/延期`
- 待归还：`确认归还`
- 已结束：`查看`

低频操作进入 `更多` 菜单：

- 详情
- 寄件
- 复制订单号
- 创建免押
- 删除

删除保持确认流程。

## 视觉约束

- 保留深绿色作为主色
- 去除大面积浅绿色背景
- 主体采用白底、细边框、轻阴影
- 绿色只用于选中态、主按钮、关键状态强调

## 风险与边界

1. 免押状态仍基于现有前端免押关联数据来源，不额外改造后端结构。
2. 本次仅调整订单履约页 UI 结构与交互入口，不改动履约业务方法。
