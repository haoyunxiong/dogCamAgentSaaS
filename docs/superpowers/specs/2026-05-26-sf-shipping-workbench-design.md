# 顺丰寄件工作台设计

日期：2026-05-26（2026-05-28 增量确认）  
状态：已确认，实施中

## 目标

在闲鱼助手桌面端新增独立的“顺丰寄件”工作台，用于国内快递查询与下单。它既支持与租赁订单关联的发货，也支持不依附租赁订单的临时寄件，减少线下重复填写和手工回写运单信息。

本期覆盖：

- 顺丰标快，`expressTypeId = 2`。
- 顺丰半日达，`expressTypeId = 263`。
- 顺丰同城费用与时效询价。
- 速运预下单、正式下单、下单结果补查、已有运单预计派送时间查询。
- 速运下单前时效和价格查询、正式订单取消、运输路由查询与清单运费查询。
- 清单运费到实际支付金额的人工修正，以及实际支付金额进入月度费用统计。

本期不覆盖：

- 港澳台或国际寄件。
- 顺丰同城正式下单，因为当前提供的文档只有费用与时效查询接口。
- 手机 H5 或飞书中的正式下单入口；桌面端流程稳定后再扩展。

## 已有基础

项目已在 `electron/main/dbManager.js` 中接入顺丰速运 OAuth token 缓存和 `/std/service` 表单请求，用于 `EXP_RECE_QUERY_DELIVERTM` 时效试算。设置页已经维护顺丰顾客编码、Secret、接口地址、寄件省市区、寄件详细地址、寄件手机号、月结卡号和货物名称。

现有租赁数据中：

- `rental_orders` 保存客户地址、计划/实际发货时间、预计送达时间、运单号和物流状态。
- `shipping_records` 保存订单物流事件。
- `OrderFulfillment.vue` 负责租赁订单履约与现有物流信息展示。

新增能力应复用现有鉴权与订单回写机制，但独立寄件不能被伪装为租赁订单。

## 页面结构

### 导航

在经营管理导航中新增 `顺丰寄件` 页面入口，与 `订单详情`、`租赁档期` 同级。

### 页面标签

页面包含三个任务标签：

1. `新建寄件`
2. `寄件记录`
3. `运单查询`

### 新建寄件

新建寄件提供两种来源：

- `独立寄件`：用户粘贴或填写收件人、手机号和国内地址。
- `关联订单`：选择已有租赁订单后自动带入收件资料、设备/型号上下文和计划发货时间；用户可在正式提交前修改寄件字段。

字段范围：

- 来源类型及可选关联订单。
- 产品类型：顺丰标快、顺丰半日达、顺丰同城询价。
- 收件姓名、手机号、省、市、区县、详细地址。
- 货物名称，默认使用设置中的租赁设备名称。
- 重量，提供合理默认值并允许修改。
- 预约揽收时间，仅在速运寄件需要时填写。
- 寄件人信息由设置带入为默认地址，页面允许当前寄件单临时编辑。
- `保存为默认寄件地址` 仅在用户主动勾选时写回设置，默认关闭；无论是否写回设置，订单均保存实际提交的寄件地址快照。

交互流程：

- 速运产品：资料齐全后防抖调用 `EXP_RECE_QUERY_DELIVERTM`，先展示预计送达与预计费用；`立即下单` -> 确认真实寄件 -> 后台自动执行预校验、正式下单与必要的结果补查 -> 显示运单、预约取件、预计送达、预计费用及筛单状态。预校验不再要求用户独立点击。
- 同城询价：填写目的地址与必要参数 -> `查询费用与时效` -> 展示预计送达、距离和费用；不提供正式下单按钮。

正式下单按钮只有在以下条件同时满足时可用：

- 产品为顺丰标快或顺丰半日达。
- 寄件方和收件方必填信息完整。
- 计划取件时间处于允许的 `08:00-20:00` 范围内。

计划取件时间提供两个快捷入口：`上午 10:00-11:00`（预约起始时间提交为 `10:00`）与 `傍晚 17:00`。用户可以手动调整日期和时间，但选择早于 `08:00` 或晚于 `20:00` 时立即清空并显示红色错误提示。

### 寄件记录

列表用于查找已发起或待处理的寄件单，默认按创建时间倒序展示。列表字段：

- 本系统寄件单号。
- 来源：独立寄件或关联订单号。
- 产品类型。
- 收件城市。
- 顺丰运单号。
- 业务状态和筛单状态。
- 预计派送时间。
- 创建时间。

点击一条记录打开详情，展示寄收件摘要、接口事件时间线、错误提示、补查、刷新预计派送、取消订单、刷新物流轨迹和查询清单运费动作。

### 运单查询

支持两种输入：

- 按本系统寄件单号或关联订单定位记录，并补查顺丰下单结果。
- 按顺丰运单号查询预计派送时间、运输轨迹与清单运费。

查询结果如果与租赁订单关联，应同步更新订单详情中的运单号或预计到货时间。

## 顺丰接口映射

### 速运通用请求

以下速运接口使用现有 OAuth access token 及 `/std/service` 表单请求封装：

- `EXP_RECE_PRE_ORDER`：正式下单前的可下单校验。
- `EXP_RECE_CREATE_ORDER`：真实创建速运订单并请求分配运单号。
- `EXP_RECE_SEARCH_ORDER_RESP`：正式下单响应未知时按客户订单号查询结果。
- `EXP_RECE_SEARCH_PROMITM`：取得运单号后查询预计派送时间。
- `EXP_RECE_QUERY_DELIVERTM`：在正式下单前按计划寄件时间、重量和寄收地址试算预计送达时间及预计费用；响应 `deliverTmDto[].fee` 作为预计费用来源。
- `EXP_RECE_UPDATE_ORDER`：正式下单成功但尚允许取消时以 `dealType = 2` 取消订单。取消成功后原客户订单号不得复用。
- `EXP_RECE_SEARCH_ROUTES`：按顺丰运单号获取揽收、运输、派送和签收路由节点。
- `EXP_RECE_QUERY_SFWAYBILL`：按顺丰运单号查询清单运费总额及费用分项。

用户已于 2026-05-28 确认上述取消、路由和清单运费接口均已上线，按生产能力交付。

速运下单仅构造中国内地字段，不传港澳台或国际寄件相关字段。

产品配置默认值：

| 产品 | 默认 `expressTypeId` |
| --- | ---: |
| 顺丰标快 | `2` |
| 顺丰半日达 | `263` |

产品编码存入配置并允许后续修改，避免账号授权变更后修改代码。

### 顺丰同城询价

`FEE_TIME_SERVICE` 与现有速运 OAuth 配置不同，使用单独的同城配置：

- `ApiUrl`
- `CompanyCode`
- `AccessCode`
- `Checkword`
- `ProjectCode`
- `ShopId`
- `ShopType`

同城凭据未配置时，产品选项可见但查询按钮禁用，并提示先完成配置。收到同城正式下单文档和凭据前，不设计同城创建运单流程。

## 支付与寄件约束

- 正式速运下单固定使用寄方付。
- 如果配置了顺丰月结卡号，先随寄方付请求提交；若顺丰返回月结不可用，自动移除月结卡并按寄付重试一次，同时在结果中提示支付方式回退。
- 没有配置月结卡号时，直接按寄付提交正式下单，不阻断寄件流程。
- 顺丰取件时间只能选择每天 `08:00-20:00`；快捷入口为 `10:00-11:00` 和 `17:00`。
- 寄件联系人需要新增设置项；寄件手机号、地址、月结卡号和货物名称沿用现有设置项。
- 正式下单前显示产品、寄件方、收件方、月结支付方式、货物和关联订单摘要，要求用户明确确认。

## 数据模型

### `sf_shipments`

保存一笔顺丰寄件业务的当前状态，适用于独立寄件及关联订单寄件。

核心字段：

- `id`：内部主键。
- `shipment_no`：系统生成的唯一寄件单号，同时作为顺丰客户订单号的基础。
- `rental_order_id`：可空，关联已有租赁订单。
- `source_type`：`standalone` 或 `rental_order`。
- `product_type`：`sf_standard`、`sf_half_day` 或 `sf_intra_city_quote`。
- `express_type_id`：速运产品编码，同城询价为空。
- 寄件人、收件人及地址快照字段。
- `cargo_name`、`weight_kg`。
- `pay_method` 和 `monthly_card_snapshot`。
- `sf_order_id`、`waybill_no`。
- `preorder_status`、`order_status`、`filter_result`、`filter_remark`。
- `promised_delivery_at`。
- `estimated_fee`：下单前接口返回的预计费用。
- `billed_fee` 和 `billed_fee_detail_json`：清单运费接口返回的总额及分项。
- `actual_paid_fee`、`actual_paid_source` 和 `actual_paid_note`：用于月度统计的最终支付金额、来源和人工修正备注；首次取得清单运费时默认等于清单总额。
- `cancelled_at`、`cancel_result_json`、`routes_result_json` 和 `fee_result_json`。
- `last_error`。
- `created_at`、`updated_at`。

保存地址和月结卡的业务快照，是为了后续核对当时提交的寄件内容；列表与日志不得完整展示敏感号码。

### `sf_shipment_events`

保存接口调用事件，支持审计、排错和防止误重复下单。

核心字段：

- `shipment_id`。
- `event_type`：`pre_order`、`create_order`、`search_order_result`、`search_promise_time`、`intra_city_quote`。
- `request_id` 和 `service_code`。
- `success`、`error_code`、`error_message`。
- `response_summary_json`：仅保存必要响应摘要，不持久化 access token 或 Secret。
- `created_at`。

### 与租赁订单同步

寄件记录关联 `rental_orders` 时：

- 获得运单号后，回写 `rental_orders.tracking_no` 和物流状态。
- 获得预计派送时间后，回写 `rental_orders.expected_arrive_at`。
- 写入或更新 `shipping_records`，使现有订单详情可直接看到最新顺丰结果。
- 运输轨迹刷新后，同步最新物流描述至现有订单详情；完整路由明细仍保存在寄件记录中。

### 与费用统计同步

- 清单运费查询成功后默认令 `actual_paid_fee = billed_fee`。
- 用户可以在详情中修改实际支付金额并填写备注；人工修正后的金额优先级高于后续清单刷新，不被自动覆盖。
- 每笔寄件单只对应一条 `expense_records.expense_type = 'shipping_sf'` 费用记录；独立寄件使用通用型号标识 `SF_STANDALONE`，关联订单优先使用订单型号。
- 月度报表聚合最终 `actual_paid_fee`，不会将预计费用或未确认的清单费用重复计入支出。

独立寄件不创建 `rental_orders`，只维护 `sf_shipments` 及事件记录。

## 后端边界与幂等

新增顺丰寄件服务模块，负责：

- 统一生成速运 `msgData`。
- 使用已有 token 获取与 `/std/service` transport。
- 标准化顺丰成功、错误、筛单状态和预计派送返回值。
- 按寄件单控制状态流转和幂等。

`dbManager.js` 负责数据库 CRUD 和订单回写，IPC 仅暴露页面所需的明确动作：

- 列表与详情。
- 创建草稿。
- 一键正式下单编排：预计送达试算、自动预校验、正式下单、必要时补查结果。
- 下单结果补查。
- 运单预计派送查询。
- 速运时效/价格预估。
- 正式订单取消。
- 运单路由查询与清单运费查询。
- 最终实际支付金额确认和费用台账同步。
- 同城费用/时效查询。

幂等规则：

- 每笔速运寄件生成并固定使用一个客户订单号。
- 已进入正式下单请求状态的寄件，不允许再次创建新顺丰订单。
- 正式下单超时、网络错误或未得到确定结果时，状态进入 `result_pending`，只允许调用 `EXP_RECE_SEARCH_ORDER_RESP` 补查。
- 补查得到运单号后再允许查询预计派送时间。

## 状态与异常显示

速运业务状态建议为：

- `draft`：填写中。
- `prechecked`：预校验成功。
- `precheck_failed`：预校验失败。
- `submitting`：正式下单请求中。
- `result_pending`：正式下单结果未确认，等待补查。
- `waybill_created`：已获得运单号。
- `manual_review`：顺丰筛单结果为人工确认或无法确定。
- `rejected`：顺丰不可收派。
- `cancelled`：顺丰订单已取消。

筛单结果显示规则：

- `filterResult = 1`：橙色，`待顺丰人工确认`。
- `filterResult = 2`：绿色，`可收派`。
- `filterResult = 3`：红色，`不可收派`，展示返回原因。
- `filterResult = 4`：橙色，`结果待确认`。

错误处理：

- 月结卡、地址、产品权限、托寄物或号码校验失败均使用显眼的红色提示块，保留输入内容用于修正。
- 取件时间超出 `08:00-20:00` 时在发起远端接口前阻止提交。
- 下单前预计送达试算失败时明确提示试算未成功；不将揽收后才可能有数据的预计派送查询作为下单完成条件。
- 只有获得可收派结果及运单号后，页面才显示已创建成功；不能把超时或筛单待确认表述为已发货。
- 预计费用、清单运费和实际支付金额需分开展示；仅实际支付金额进入月度费用统计。
- 取消操作必须二次确认；已取消订单不可再次下单，已产生不可取消状态时展示顺丰错误而不伪造取消成功。
- 秘钥、access token 和完整月结卡号不得写入界面日志或事件摘要。

## 配置增量

在设置的顺丰配置区域补充：

- 寄件联系人。
- 顺丰标快产品编码，默认 `2`。
- 顺丰半日达产品编码，默认 `263`。
- 同城 `ApiUrl`、`CompanyCode`、`AccessCode`、`Checkword`、`ProjectCode`、`ShopId`、`ShopType`。

现有速运配置继续复用：

- `SF_APP_ID`、`SF_APP_KEY`。
- `SF_TOKEN_API`、`SF_PREORDER_API`。
- 寄件省市区、详细地址和手机号。
- 月结卡号和货物名称。

## 验证

自动化验证：

- 产品编码默认值及配置覆盖。
- 无月结卡号时使用寄付正式下单。
- 收寄地址及国内件报文构造。
- 固定客户订单号和重复下单保护。
- `filterResult` 状态映射。
- 正式下单未知结果进入补查流程。
- 预计派送时间解析并回写关联租赁订单。
- 独立寄件不创建租赁订单。
- 同城未配置时禁用查询、配置后正确映射费用和预计送达返回值。
- 下单前预计费用解析与刷新。
- 取消报文、取消状态和重复操作保护。
- 路由节点解析、最新物流状态同步。
- 清单运费分项/总额解析，默认实付入账及手工修正不被后续刷新覆盖。

界面验证：

- 桌面宽窗口与窄窗口下三个标签页布局清晰，无字段重叠。
- 独立寄件与关联订单带入两条路径可完成预校验流程。
- 错误、待确认和成功状态清晰区分，且不只依赖颜色。

真实接口验证：

- 预校验与查询可以使用真实配置验证。
- 正式创建运单会产生真实寄件任务，仅在用户明确确认测试寄件时执行一笔。
