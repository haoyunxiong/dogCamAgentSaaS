# 租赁核心业务流 V2 桌面端 UI / 交互设计规格

本文档定义桌面端核心租赁流程的页面、抽屉、弹窗和交互口径。本文是设计规格，不代表本轮实现。所有真实写操作仍必须遵守 Phase 04 safeOps 和 external gateway 边界。

## 1. 设计目标

桌面端是完整管理后台，目标是支持客服、仓库、店长、财务和店主完成高频运营工作。

设计原则：

- 高密度但可读；
- 工作台优先展示待办和风险；
- 核心业务页以表格、筛选、详情抽屉为主；
- 不做营销页；
- 不用重渐变、玻璃拟态、装饰动画；
- 所有关键动作必须有状态、原因、影响预览和下一步提示；
- 查询阶段不采集完整客户隐私；
- 真实顺丰和真实免押入口默认 disabled 或 preview-only。

## 2. 桌面端信息架构

建议主导航：

| 一级导航 | 主要用途 |
|---|---|
| 工作台 | 今日待办、风险提醒、下一步动作、快捷入口 |
| 档期中心 | 快速查档期、型号余量、单机排期、临时锁定 |
| 订单中心 | 订单列表、订单详情、创建订单四步向导 |
| 设备管理 | 型号、单机、价格残值、维修停用 |
| 物流管理 | 发出物流、归还物流、顺丰预览、手动关联 |
| 免押管理 | 创建免押、关联订单、审核状态、完结确认 |
| 报表中心 | 收入、设备利用率、渠道、履约、异常 |
| 设置 | 门店、员工、权限、配置、外部 gateway 状态 |

## 3. 通用交互模式

### 3.1 页面结构

业务页面建议统一结构：

```text
[页面标题 + 核心动作按钮]
[关键指标 / 风险提示 / 当前筛选摘要]
[筛选栏]
[主表格 / 排期 / 列表]
[右侧详情 Drawer]
```

### 3.2 Drawer 规则

- Drawer 用于详情查看、轻量编辑、创建业务对象。
- Drawer 宽度建议 520-720px。
- 危险动作放在 Drawer 底部，必须二次确认。
- 外部真实接口按钮默认禁用，展示“当前 external real disabled”。
- Drawer 内不得隐藏关键状态，例如免押、物流、验机、档期冲突。

### 3.3 Modal 规则

- Modal 用于二次确认、归还验收、审批、强制覆盖。
- Modal 必须展示影响范围和原因输入。
- 不能在 Modal 内堆完整业务流程，复杂流程进入 Drawer 或向导。

### 3.4 下一步动作卡

订单、工作台、档期、免押、物流都应使用统一下一步动作卡：

| 字段 | 说明 |
|---|---|
| 动作标题 | 例如“创建免押单”“创建发出物流”“开始验机” |
| 触发原因 | 哪些状态导致此动作 |
| 风险等级 | 正常 / 风险 / 阻断 |
| 主按钮 | 推荐下一步 |
| 次按钮 | 查看详情、人工覆盖、忽略 |
| 所需权限 | actor role |

## 4. 档期中心

### 4.1 快速查档期

位置：档期中心顶部固定区域。

字段：

- 设备型号；
- 租期开始；
- 租期结束；
- 模糊目的地区域；
- 发货门店。

不得出现：

- 姓名；
- 手机号；
- 详细门牌地址。

交互：

1. 员工输入模糊地址。
2. 系统解析省 / 市 / 区县 / 镇乡街道。
3. 如果只到城市，阻止查询并提示选择区县。
4. 如果候选不唯一，展示候选区县选择。
5. 查询后进入结果页。

### 4.2 查询结果页

结果页分四个区域：

| 区域 | 内容 |
|---|---|
| 查询摘要 | 型号、租期、发货门店、目的地区县、解析置信度 |
| 推荐结论 | 安全可用 / 风险可用 / 需人工确认 / 无档期 |
| 推荐单机 | 设备编号、所属门店、当前状态、前一单回仓、下一预约 |
| 时间计算 | 10:00 发货、18:00 发货、最晚发货、必须送达日、缓冲、重新可租时间 |

关键按钮：

- 创建临时锁定；
- 换一台推荐；
- 查看单机排期；
- 复制给客户的话术；
- 无档期时推荐其它型号 / 门店 / 租期。

### 4.3 临时锁定倒计时

锁定成功后在结果页和顶部工作条显示倒计时。

显示字段：

- 剩余时间；
- 锁定单机；
- 锁定租期；
- 完整物流占用窗口；
- 操作员工；
- 到期时间；
- 风险提示。

动作：

- 转订单；
- 延长锁定；
- 手动释放；
- 查看冲突。

倒计时低于 60 秒时使用高亮提示，但不自动弹窗打断操作。

### 4.4 型号余量

展示方式：

- 左侧型号列表或顶部型号卡；
- 显示总库存、可用、风险可用、已锁定、已预约、维修 / 下架；
- 支持按门店筛选；
- 点击型号后联动单机排期。

### 4.5 单机排期

展示方式：

- 行：单机；
- 列：日期；
- 区块：发出物流、租赁中、归还物流、缓冲、维修、临时锁定；
- 鼠标悬停展示订单号、客户脱敏信息、来源、风险；
- 点击区块打开订单或锁定详情。

颜色语义：

| 类型 | 表达 |
|---|---|
| 安全可用 | 低饱和绿色或中性色可用标识 |
| 风险可用 | 黄色风险标识 |
| 需人工确认 | 橙色标识 |
| 无档期 / 冲突 | 红色阻断标识 |
| 临时锁定 | 蓝色倒计时标识 |

### 4.6 临时锁定列表

字段：

- 锁定编号；
- 型号；
- 单机；
- 租期；
- 目的地区县；
- 发货门店；
- 操作员工；
- 剩余时间；
- 状态；
- 下一步动作。

筛选：

- 锁定中；
- 即将过期；
- 已转订单；
- 已过期释放；
- 手动释放；
- 员工；
- 门店。

## 5. 创建订单四步向导

### 5.1 向导入口

入口：

- 临时锁定结果页的“转订单”；
- 档期中心“复用最近查询结果创建”；
- 订单中心“新建订单”；
- 客户沟通页面后续可接入，但不作为当前阶段前提。

### 5.2 步骤 1：档期与设备

目的：确认设备和完整占用窗口。

字段：

- 型号；
- 租期；
- 发货门店；
- 目的地区县；
- 推荐单机；
- 临时锁定状态；
- 10:00 / 18:00 发货时效；
- 前一单回仓和缓冲；
- 风险分类。

动作：

- 重新查档期；
- 更换推荐单机；
- 延长锁定；
- 人工覆盖冲突。

人工覆盖必须输入原因，且按权限要求审批。

### 5.3 步骤 2：客户与地址

目的：创建订单阶段才采集完整客户信息。

输入方式：

- 粘贴完整收货信息；
- 手动填写姓名、手机号、省市区县、详细地址；
- 选择已有客户。

自动解析：

- 姓名；
- 手机号；
- 省；
- 市；
- 区县；
- 详细地址。

客户匹配：

- 手机号强匹配；
- 姓名 + 区县弱匹配；
- 手机号缺失时标记“资料不完整”；
- 多候选时由员工选择。

隐私：

- 列表和日志默认脱敏手机号和详细地址；
- 只有订单详情和履约 Drawer 可显示完整信息，且需权限控制。

### 5.4 步骤 3：价格与押金

目的：确认租金、押金 / 免押金额和价格快照。

展示：

- 型号价格规则；
- 租期天数；
- 基础租金；
- 续租规则；
- 优惠；
- 最终租金；
- 型号残值；
- 默认押金 / 免押金额；
- 是否需要免押；
- 手动调整区域。

手动调整：

- 修改租金必须填原因；
- 修改押金 / 免押金额时显示三个选项：
  - 仅本单使用；
  - 同步更新该型号残值；
  - 提交管理员审核。

### 5.5 步骤 4：确认创建

展示最终摘要：

- 订单渠道；
- 客户和地址；
- 型号和单机；
- 租期；
- 完整物流占用窗口；
- 价格快照；
- 押金 / 免押；
- 发货门店；
- 风险提示；
- 下一步动作。

创建按钮：

- 默认需要二次确认；
- 重复点击必须幂等；
- 成功后进入订单详情。

创建成功后的下一步动作：

- 创建免押；
- 收押金；
- 创建发出物流；
- 补全资料；
- 人工确认风险。

## 6. 订单详情

订单详情使用右侧大 Drawer 或独立详情页，桌面端建议可从列表打开 Drawer，再进入完整详情。

### 6.1 流程进度条

进度条节点：

```text
临时锁定 -> 待确认 -> 押金/免押 -> 待发货 -> 已发货 -> 租赁中 -> 待归还 -> 归还运输中 -> 待验机 -> 已完成
```

异常、取消作为分支状态展示，不挤进主线进度。

### 6.2 下一步动作卡

订单详情顶部必须展示一个主下一步动作。

示例：

| 当前条件 | 下一步动作 |
|---|---|
| 未创建免押 | 创建免押单 |
| 免押已通过但未发货 | 创建发出物流 |
| 已发货且预计送达异常 | 跟进物流 |
| 租期明天结束 | 提醒客户归还 |
| 已归还未验机 | 开始验机 |
| 验机正常且免押未完结 | 完结免押 |

### 6.3 信息区块

订单详情必须包含：

| 区块 | 内容 |
|---|---|
| 客户与地址 | 姓名、手机号、完整收货地址、客户风险、隐私脱敏规则 |
| 设备与档期 | 型号、单机、所属门店、完整占用窗口、单机排期 |
| 价格与押金 | 租金、押金 / 免押金额、价格快照、调整原因 |
| 免押 | 免押单号、状态、金额、创建入口、关联入口、完结入口 |
| 发出物流 | 寄件方式、运单号、顺丰预览、发货状态 |
| 归还物流 | 寄回方式、运单号、线下归还、归还状态 |
| 验机 | 验机结果、设备状态影响、免押完结判断 |
| 时间线 | 创建、锁定、改价、免押、发货、归还、验机、异常 |
| 内部备注 | 员工备注、风险备注、只读 / 可写权限 |

## 7. 免押页面与 Drawer

### 7.1 免押创建 Drawer

入口：

- 订单详情；
- 免押管理；
- 工作台下一步动作。

字段：

- 订单号；
- 手机号；
- 型号；
- 租期；
- 租金；
- 型号残值；
- 免押金额；
- 客户姓名；
- 门店；
- 操作员工。

交互：

- 从订单进入时自动填充；
- 手动创建时可先搜索订单；
- 外部 real disabled 时只允许 preview；
- 展示脱敏 payload；
- 创建真实免押前必须通过 external gateway。

### 7.2 免押关联订单 Drawer

用于处理未关联免押单。

布局：

```text
[免押单摘要]
[候选订单筛选：手机号 / 型号 / 租期 / 金额 / 时间]
[候选订单列表 + 匹配分]
[匹配原因]
[确认关联按钮]
```

候选列表字段：

- 订单号；
- 客户手机号脱敏；
- 型号；
- 租期；
- 租金；
- 免押金额；
- 创建时间；
- 匹配分；
- 风险原因。

低置信度候选必须要求人工确认原因。

### 7.3 完结免押确认

完结免押前必须检查：

- 订单已归还；
- 验机完成；
- 无损坏；
- 无欠费；
- 无逾期争议；
- 归还物流已确认；
- 设备状态允许入库。

Modal 显示：

- 检查项列表；
- 不满足项；
- 外部 real disabled 状态；
- 操作人；
- 审批人；
- 原因。

## 8. 物流页面与 Drawer

### 8.1 发出物流 Drawer

入口：

- 订单详情；
- 物流管理；
- 工作台下一步动作。

从订单进入时自动带入：

- 订单号；
- 客户收货信息；
- 设备型号；
- 具体设备；
- 发货门店；
- 默认寄件人信息；
- 推荐顺丰产品；
- 最晚发货时间。

字段：

- 物流类型：发出物流；
- 承运商；
- 服务类型：普通快递 / 顺丰标快 / 半日达 / 同城急送 / 其他；
- 发货门店；
- 寄件人；
- 收件人；
- 预计发货时间；
- 运单号；
- 备注。

动作：

- 保存本地记录；
- 顺丰预览；
- 标记已发货；
- 取消。

真实顺丰下单按钮默认 disabled。

### 8.2 归还物流 Drawer

字段：

- 物流类型：归还物流；
- 归还方式：上传寄回单号 / 不上传单号 / 线下归还 / 手动标记已归还；
- 承运商；
- 寄回单号；
- 预计到仓时间；
- 归还门店；
- 接收员工；
- 原因说明。

动作：

- 记录归还物流；
- 标记已回仓；
- 进入归还验收；
- 异常处理。

### 8.3 物流关联订单 Drawer

用于手动创建物流后关联订单。

候选规则：

- 手机号；
- 地址区县；
- 型号；
- 租期；
- 发货 / 归还方向；
- 创建时间；
- 运单号备注。

低置信度必须人工确认原因。

### 8.4 顺丰预览 Drawer

展示：

- external gateway mode；
- 是否会真实调用；
- 脱敏 payload；
- 预计时效；
- 预计费用；
- 风险；
- idempotencyKey；
- confirm 要求。

real mode disabled 时：

- 主按钮显示禁用；
- 文案说明“当前仅支持 mock / sandbox preview，不会真实下单”；
- 不显示完整手机号、姓名、地址和密钥。

### 8.5 归还验收 Modal

字段：

- 订单号；
- 设备；
- 归还时间；
- 配件清单；
- 外观状态；
- 功能状态；
- 电池 / 存储卡 / 充电器；
- 照片或备注入口；
- 验机结果。

验机结果选项：

- 正常入库；
- 需要清洁；
- 需要维修；
- 设备异常；
- 丢失 / 损坏。

提交后展示影响：

- 设备状态；
- 订单主状态；
- 免押是否可完结；
- 后续档期是否释放；
- 客户风险。

## 10. 工作台

### 10.1 今日待办

待办类型：

- 即将过期临时锁定；
- 待创建免押；
- 待收押金；
- 待发货；
- 发货时效风险；
- 租期即将结束；
- 待归还；
- 归还运输异常；
- 待验机；
- 免押可完结；
- 设备异常；
- 待审批。

### 10.2 风险提醒

风险类型：

- 档期冲突；
- 顺丰时效不确定；
- 前一单回仓风险；
- 客户资料不完整；
- 免押未关联订单；
- 押金金额异常；
- 设备残值被人工修改；
- 已发货订单取消申请；
- 归还逾期；
- 验机异常。

### 10.3 下一步动作

工作台按优先级展示下一步动作：

1. 阻断履约的异常；
2. 今天必须处理的发货和归还；
3. 即将过期锁定；
4. 待押金 / 免押；
5. 待验机；
6. 待审批；
7. 可优化的低风险事项。

### 10.4 快捷入口

快捷入口：

- 快速查档期；
- 新建订单；
- 创建免押；
- 创建发出物流；
- 录入归还；
- 开始验机；
- 新建设备；
- 查看风险订单。

## 11. 状态与视觉规范

### 11.1 状态表达

状态必须同时使用文本和颜色，不只靠颜色区分。

建议状态组件：

- `StatusBadge`：主状态；
- `FilterChip`：筛选状态；
- `MetricCard`：指标；
- `ScheduleBlock`：档期块；
- `OrderCard`：订单摘要；
- `DeviceCard`：设备摘要。

### 11.2 信息密度

- 表格默认显示 10-15 列以内；
- 次要字段进 Drawer；
- 高风险状态在列表中前置；
- 手机号和详细地址默认脱敏；
- 员工重复操作不应需要跨多个页面跳转。

## 12. 数据与安全边界

### 12.1 查询阶段隐私

查询档期阶段不得展示姓名、手机号和详细门牌地址输入框。

### 12.2 外部 API

顺丰和免押：

- 默认 disabled；
- mock / sandbox preview 不发送真实请求；
- real mode 需要单独确认；
- payload 脱敏；
- 所有真实写操作必须 safeOps + external gateway。

### 12.3 人工覆盖

以下 UI 动作必须收集原因：

- 强制忽略档期冲突；
- 大幅改租金；
- 改押金 / 免押金额；
- 修改型号残值；
- 手动标记已归还；
- 手动完结异常订单；
- 取消已发货订单。
