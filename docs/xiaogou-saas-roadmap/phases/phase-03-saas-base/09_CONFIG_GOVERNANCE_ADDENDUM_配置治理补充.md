# Phase 03 补充：配置中心与敏感配置治理

## 1. 补充原因

Phase 03 的核心是 SaaS 底座，包括登录、商户、门店、员工、权限和数据隔离。配置治理与这些内容强相关，因此应纳入 Phase 03 的需求范围。

## 2. Phase 03 中配置治理要解决什么

- 确认当前配置来源；
- 区分系统默认配置、本地运行配置、商户业务配置、敏感凭证；
- 防止真实密钥进入 Git；
- 为多商户配置隔离做准备；
- 设计 ConfigService / CredentialService；
- 规划 merchant_settings、store_settings 等表；
- 不破坏现有功能。

## 3. Phase 03 配置治理任务

### 3.1 只读审计

输出：

```text
docs/xiaogou-saas-roadmap/phases/phase-03-saas-base/artifacts/config-source-audit.md
```

### 3.2 数据归属判断

判断每个配置未来归属：

```text
代码默认配置
本地运行配置
MySQL 商户配置
加密凭证库
```

### 3.3 敏感凭证治理

要求：

- 不写死真实敏感值；
- 不提交本地敏感文件；
- 前端不接触真实密钥；
- 后续通过 CredentialService 读取。

### 3.4 开发规划

本阶段不一定立即实现全部配置中心，但必须输出：

- 配置服务设计；
- 数据表建议；
- 迁移计划；
- 回滚计划；
- 验收标准。

## 4. Phase 03 不做什么

- 不直接删除旧配置；
- 不一次性重构所有配置读取；
- 不在没有迁移方案时改数据库；
- 不暴露真实敏感值；
- 不破坏旧功能。

## 5. 验收补充

Phase 03 若执行配置治理，必须满足：

- [ ] 配置来源审计完成；
- [ ] 敏感命中复核完成；
- [ ] 本地敏感文件被 Git 忽略；
- [ ] 商户配置未来 DB 归属明确；
- [ ] 敏感凭证加密存储方案明确；
- [ ] 有迁移和回滚计划；
- [ ] 旧功能未受影响。
