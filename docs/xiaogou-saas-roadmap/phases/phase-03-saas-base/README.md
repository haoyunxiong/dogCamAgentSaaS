# Phase 03：SaaS 底座：登录、商户、门店、员工、权限、数据隔离

## 阶段目标

把系统从单商户内部工具改造成可多商户使用的基础架构。

## 本阶段 Codex 做什么

- 设计登录与商户模型
- 增加 merchant_id/store_id
- 设计员工/角色/权限
- 迁移旧数据到默认商户
- 实现登录保护
- 输出备份和回滚方案

## 本阶段你做什么

- 确认登录方式
- 确认第一版角色
- 确认默认商户信息
- 确认旧数据迁移方案
- 测试登录后核心流程

## 本阶段交付物

- 数据库迁移方案
- 登录系统
- 商户/门店/员工基础模块
- 数据隔离验证报告

## 进入下一阶段条件

- 可登录
- 旧数据可见
- 核心数据绑定 merchant_id
- 未登录不能进业务页
- 完成备份和回滚验证

## 执行原则

- 先计划，后执行。
- 不跨阶段。
- 不破坏现有可用流程。
- 结果写入本阶段 `artifacts/`。
