# Phase UI-V2 Demo 产品设计简报

## 设计目标

摆脱旧系统页面结构限制，重新定义“小狗相机助手商户版”的新产品体验。

## 输入来源

- Phase 01 产品定位与信息架构
- Phase 02A Product Design 输入
- Phase 02B Figma Design System
- 现有租赁业务流程和真实使用场景

## 设计原则

- 先定体验，再迁移能力。
- PC 端是完整管理后台。
- Mobile 端是移动运营工作台。
- 信息密度高，但要可扫读。
- 状态和下一步动作优先于装饰。
- 小狗 IP 只作为轻量品牌元素。
- 闲鱼只作为 legacy channel，不进入产品核心。

## 必须避免

- 旧 UI 美化版。
- 被旧路由、旧页面、旧布局限制。
- 把旧系统业务逻辑提前接进 Demo。
- 在 Demo 阶段处理数据库、Electron、Python 或真实 API。
