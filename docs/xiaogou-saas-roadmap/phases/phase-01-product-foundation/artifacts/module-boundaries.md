# Phase 01 模块边界说明

确认日期：2026-06-12

## 当前主线模块

| 模块 | 当前边界 | 不包含 |
|---|---|---|
| 工作台 | 今日待办、经营概览、异常提醒、手机端快捷入口 | 不做复杂报表和权限系统 |
| 询单中心 | 询单记录、查档期、报价、转订单 | 不把闲鱼会话作为唯一入口 |
| 订单中心 | 创建订单、订单列表、订单详情、履约流转 | 不在 Phase 01 实现状态机 |
| 档期中心 | 设备占用、空闲查询、租期和缓冲期概念 | 不在 Phase 01 改算法 |
| 设备中心 | 设备型号、单台设备、设备状态、资产管理 | 不在 Phase 01 改 schema |
| 客户中心 | 客户资料、订单历史、沟通记录入口 | 不在 Phase 01 开发 CRM |
| 免押管理 | 免押申请、押金状态、规则查看 | 不在 Phase 01 改接口或资金逻辑 |
| 物流发货 | 发货、运单、物流追踪、批量履约入口 | 不在 Phase 01 改顺丰逻辑 |
| 财务结算 | 租金、押金、退款、成本、结算口径入口 | 不在 Phase 01 改财务计算 |
| 报表中心 | 收入、设备效率、渠道、经营分析入口 | 不在 Phase 01 深化报表 |
| 设置中心 | 基础设置、渠道配置、商户配置预留 | 不在 Phase 01 开发配置中心 |

## 保留能力边界

以下能力必须保留，但后续可以去闲鱼化、渠道化、SaaS 化：

1. 查档期。
2. 发档期 / 报价。
3. 创建订单。
4. 订单履约。
5. 待发货 / 进行中 / 待归还 / 逾期 / 完结。
6. 免押 / 押金。
7. 顺丰发货和物流追踪。
8. 归还 / 验机 / 完结。
9. 设备型号 / 单台设备。
10. 设备状态。
11. 报表统计。
12. 费用统计。
13. 配置能力。
14. 日志和运行状态。
15. 当前启动流程。
16. 现有有效数据保护。
17. 店铺 / 门店维度。
18. 知识库。
19. 商品 / 型号绑定中与租赁业务相关的能力。

保留的是业务能力，不是旧命名、旧 UI 或旧闲鱼实现方式。

## 配置治理边界

CR-0001_配置中心与敏感配置治理 已纳入 Phase 03 Backlog。

Phase 01 不执行：

- 配置审计。
- ConfigService / CredentialService 开发。
- 数据库迁移。
- 敏感配置迁移。
- 业务代码修改。

Phase 01 只在模块边界中说明配置治理属于 Phase 03 SaaS 底座阶段。

## 商户协作边界

商户之间调货、代发、二手设备交易、商户信用、分润结算、平台担保等能力，只作为长期方向预留。

Phase 01 不做：

- 不进入核心模块。
- 不进入手机端底部导航。
- 不进入 PC 端主导航。
- 不写交易闭环。
- 不写支付和分润规则。
