# Phase 01 核心业务流程确认

确认日期：2026-06-12

## 核心流程

Phase 01 固定核心业务流程为：

```text
询单
→ 查档期
→ 报价 / 发档期
→ 创建订单
→ 创建免押 / 押金
→ 发货
→ 物流跟踪
→ 到期提醒
→ 归还验机
→ 完结订单
→ 数据沉淀 / 报表分析
```

## 流程说明

| 步骤 | 业务含义 | 主要参与角色 | Phase 01 处理 |
|---|---|---|---|
| 询单 | 客户提出租赁需求 | 运营 | 固定主流程 |
| 查档期 | 判断设备和日期是否可租 | 运营、店长 | 固定主流程 |
| 报价 / 发档期 | 给客户发送可租方案和价格 | 运营 | 固定主流程 |
| 创建订单 | 将客户需求转成订单 | 运营、店长 | 固定主流程 |
| 创建免押 / 押金 | 处理风控和资金约束 | 运营、财务 | 固定主流程 |
| 发货 | 打包、寄件、记录运单 | 发货员 | 固定主流程 |
| 物流跟踪 | 跟进配送状态和异常 | 发货员、运营 | 固定主流程 |
| 到期提醒 | 提醒归还、续租或异常处理 | 运营、店长 | 固定主流程 |
| 归还验机 | 收回设备并检查状态 | 发货员、店长 | 固定主流程 |
| 完结订单 | 完成订单闭环和资金处理 | 财务、运营 | 固定主流程 |
| 数据沉淀 / 报表分析 | 沉淀经营、设备、渠道和效率数据 | 老板、店长、财务 | 固定主流程 |

## 当前阶段不实现

Phase 01 只固定流程，不实现：

- 订单状态机。
- 档期冲突算法。
- 免押/押金接口改造。
- 物流流程改造。
- 归还验机数据模型。
- 报表计算逻辑。
- 员工权限控制。
