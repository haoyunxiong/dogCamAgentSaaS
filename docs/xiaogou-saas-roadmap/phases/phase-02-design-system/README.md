# Phase 02：设计系统与 Figma 基础组件

## 阶段目标

建立稳定 Design System 和 Figma 组件库，避免后续 UI 反复推翻。

## 本阶段 Codex 做什么

- 通过 Figma MCP 创建设计系统
- 创建颜色变量/字体样式/组件
- 创建移动端核心页面初稿
- 创建 PC 端后台基础框架
- 输出设计验收报告

## 本阶段你做什么

- 提供参考产品
- 确认主色和品牌风格
- 审核 Figma 初稿
- 确认以后不大幅换风格

## 本阶段交付物

- Figma 设计系统
- 组件库
- 移动端核心页面初稿
- PC 端框架初稿

## 进入下一阶段条件

- 颜色/字体/组件确认
- 订单卡和档期块确认
- 移动端工作台初稿确认
- Figma 文件可编辑
- 设计系统可复用

## 执行原则

- 先计划，后执行。
- 不跨阶段。
- 不破坏现有可用流程。
- 结果写入本阶段 `artifacts/`。
