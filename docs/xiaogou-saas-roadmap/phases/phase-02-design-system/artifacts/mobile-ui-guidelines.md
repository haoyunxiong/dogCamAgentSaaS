# Mobile UI Guidelines / 手机端 UI 指南

## 1. 手机端定位

手机端是移动运营工作台，不是 PC 后台缩小版。

主要服务场景：

- 店长路上快速看待办；
- 运营快速查订单；
- 发货员确认发货；
- 员工快速查档期；
- 快速查设备状态。

## 2. 固定底部导航

```text
工作台 / 订单 / 档期 / 设备 / 我的
```

不进入底部导航的功能：

- 客户管理；
- 物流发货；
- 报表中心；
- 财务中心；
- 系统设置；
- 免押管理。

这些功能放入：

- 工作台快捷操作；
- 我的 / 更多工具；
- 订单详情关联入口。

## 3. 页面原则

- 一屏优先展示最重要任务；
- 卡片只展示必要字段；
- 复杂筛选使用 Bottom Sheet；
- 详情页按分组卡片组织；
- 主要操作固定在底部；
- 危险操作进入“更多操作”。

## 4. 手机端页面说明

### 工作台

首屏必须包含：

- 今日待处理核心指标；
- 快捷操作；
- 今日待办；
- 风险提醒。

### 订单

使用订单卡，不使用表格。每张订单卡显示：

- 订单号；
- 客户；
- 设备；
- 租期；
- 金额；
- 状态；
- 一个主操作。

### 档期

使用周视图 + 设备热力条，不展示完整月表格。

### 设备

使用设备卡，突出设备状态：在租、可用、维修、异常。

### 我的

收纳低频功能与设置，不承载高频操作。
