# Frontend Implementation Brief / 前端落地简报

## 1. 总体策略

前端落地分两步：

```text
先 UI 骨架与组件体系
再接真实业务功能
```

第一阶段可以使用 mock 数据还原 UI，不允许为了 UI 改动核心业务逻辑。

## 2. 推荐实现顺序

```text
Step 1: Design Tokens 与全局样式
Step 2: Base Components
Step 3: PC AppShell / Sidebar / Topbar
Step 4: PC 订单中心静态 UI
Step 5: PC 工作台 / 档期 / 设备 / 客户 / 物流 / 报表 / 设置
Step 6: Mobile AppShell / Bottom Nav
Step 7: Mobile 工作台 / 订单 / 订单详情 / 档期 / 设备 / 我的
Step 8: 空状态 / 加载态 / 异常态
Step 9: 接现有数据
Step 10: 新功能开发
```

## 3. PC 端实现重点

- Sidebar 对齐必须严格统一；
- 表格页采用右侧 Drawer；
- KPI 卡、状态筛选、筛选栏、批量操作、分页统一；
- 页面高频动作必须靠近当前任务；
- 详情 Drawer 底部固定主要操作。

## 4. 手机端实现重点

- 使用 5 个底部 Tab；
- 首屏只展示高频任务；
- 订单列表使用卡片；
- 档期使用小型热力条/周视图；
- 低频功能收进“我的 / 更多工具”；
- 详情页使用分组卡片与底部固定操作。

## 5. 可先 Mock 的内容

- KPI 数据；
- 订单列表；
- 设备列表；
- 客户列表；
- 报表图表；
- 档期热力图；
- 右侧 Drawer 数据。

## 6. 后续接真实数据

等 UI 结构稳定后，再逐页接入现有业务数据：

```text
订单中心 → 订单列表/状态/详情/发货
档期中心 → 设备/型号档期
设备中心 → 设备状态/详情/维修/调拨
物流发货 → 运单/面单/发货状态
客户管理 → 客户列表/客户详情
报表中心 → 统计接口
设置中心 → Phase 03 后接配置治理
```
