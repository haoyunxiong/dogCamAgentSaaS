# Desktop Product IA Hardening

Phase 04 本轮目标是加固 UI-V2 桌面端产品语义、信息架构、数据口径和交互闭环。范围只包含桌面端业务页、UI-V2 adapter / mapper 和当前阶段文档；不改 mobile，不改 DB schema，不开启外部 real API。

## 通用约束

- 页面围绕运营任务组织，不直接围绕数据库表展示。
- 真实数据缺失时显示 `未填写`、`暂无记录`、`无法判断`，不使用随机数或假指标填充。
- 普通业务页不展示 raw operationType、provider key、safeOps 内部字段、数据库字段名。
- 所有列表默认分页，每页 20 至 50 条；筛选、Tab、按钮必须有反馈。
- 外部顺丰、免押、闲鱼保持 mock / sandbox preview-only，real disabled。

## Orders

- 核心实体：当前运营订单、历史归档订单。
- 默认视图：`进行中`，隐藏大量已完成 / 已取消历史订单到 `历史归档`。
- 数据源：`rental_orders` 及订单详情只读 IPC。
- 指标口径：按订单当前阶段统计待发货、待归还、异常、在租、历史归档；金额为订单基础金额汇总。
- 主操作：内部备注安全更新。
- 次操作：顺丰预览、闲鱼同步预览、编辑 / 状态 dry-run 预览。
- 空态：`暂无需要处理的订单` 或 `历史归档暂无记录`。
- 缺失表达：客户、手机号、设备、门店、租期分别显示未填写 / 暂无记录。
- Drawer：客户、设备、租期、费用、物流、免押、时间线、安全操作。

## Devices

- 核心实体：设备型号库存组、单机资产。
- 默认视图：`型号视图`。
- 数据源：`schedule_units` effective status 只读结果。
- 指标口径：按型号汇总总台数、可租、在租、已预订、维修、停用；近 30 天收入仅在有可靠订单数据时显示，否则 `无法判断`。
- 主操作：`device.basic.update` 受控基础字段更新。
- 次操作：维护 / 停用 / 删除仅保留预览或禁用。
- 空态：`暂无匹配型号` / `暂无匹配设备`。
- 缺失表达：残值、最近维护、下一预约无数据时显示无法判断 / 暂无记录。
- Drawer：型号组展开单机清单；单机 Drawer 展示资产详情、占用、备注和安全操作。

## Schedule

- 核心实体：型号每日余量、单机排期 block。
- 默认视图：`型号余量总览`。
- 数据源：`schedule_units`、`schedule_blocks`、月度 overview 只读 IPC。
- 指标口径：每日总库存、占用数、可用数、冲突数；单机视图按真实 block 显示占用 / 锁定 / 维修 / 取消。
- 主操作：`schedule.block.create`、`schedule.block.cancel`。
- 次操作：批量锁库继续 disabled / preview-only。
- 空态：`暂无档期记录`。
- 缺失表达：无订单、无备注、无日期时显示暂无记录 / 未填写日期。
- Drawer：型号日期余量或单机 block 详情，包含冲突说明和安全操作。

## Customers

- 核心实体：从订单、询单、免押缓存派生的客户资料。
- 默认视图：派生客户总览。
- 数据源：`rental_orders`、`inquiry_events`、免押本地缓存。
- 指标口径：手机号标准化优先去重；缺手机号时以姓名 + 城市弱去重；风险标注为系统派生。
- 主操作：查看客户履约上下文。
- 次操作：联系 / 标记只保留页面反馈，不写业务表。
- 空态：`暂无可派生客户`。
- 缺失表达：城市、电话、最近订单、免押记录分别显示未填写 / 暂无记录。
- Drawer：订单历史、设备偏好、免押记录、风险依据。

## Logistics

- 核心实体：发货任务、物流记录。
- 默认视图：`待创建运单`。
- 数据源：发货任务来自需要寄出的订单；物流记录来自 `shipping_records`、`sf_shipments`、`sf_shipment_events`。
- 指标口径：任务和记录分开统计，禁止用订单总数冒充物流记录数。
- 主操作：`logistics.local_record.create`。
- 次操作：顺丰 mock / sandbox preview，real disabled。
- 空态：`暂无发货任务` / `暂无物流记录`。
- 缺失表达：运单号、轨迹、预计送达、风险显示暂无记录 / 无法判断。
- Drawer：任务详情、物流记录详情、本地记录创建、顺丰预览。

## Deposit

- 核心实体：免押审核 / 创建 / 完结的本地缓存与接口同步快照。
- 默认视图：`待审核`。
- 数据源：免押接口同步结果、本地缓存降级。
- 指标口径：明确接口同步状态、最后同步时间、本地缓存状态、订单关联状态、风险原因和状态时间线。
- 主操作：免押 create / finish mock / sandbox preview。
- 次操作：刷新同步；real disabled。
- 空态：`暂无免押记录`。
- 缺失表达：同步时间、外部单号、风险原因缺失时显示暂无记录 / 无法判断。
- Drawer：数据来源、订单关联、风险依据、状态时间线、安全预览。

## Reports

- 核心实体：经营统计周期。
- 默认视图：最近有数据周期。
- 数据源：订单、设备、档期、免押缓存只读聚合。
- 指标口径：累计指标和区间指标分开；展示数据库覆盖日期范围；无数据时不绘制全 0 趋势。
- 主操作：切换门店 / 时间 / 渠道 / 设备类型筛选。
- 次操作：导出按钮只给反馈，不做导出。
- 空态：`该时间范围暂无数据`。
- 缺失表达：不可计算利用率标注估算或无法判断。

## Dashboard

- 核心实体：今日运营待办、风险提醒、快捷操作、运营概览。
- 默认视图：运营优先，技术状态折叠到系统状态区。
- 数据源：订单、设备、档期、客户、health / config adapter。
- 指标口径：今日待处理、待发货、待归还、异常订单、当前在租、未来 7 天档期风险；累计数据单独展示。
- 主操作：进入订单、发货、档期、设置。
- 空态：今日无新订单时显示运营空态，不混入累计订单。

## Settings

- 核心实体：七个设置模块。
- 默认视图：`商户与门店`。
- 数据源：`stores`、`config` 非敏感配置、health check、actor context。
- 七个模块：商户与门店、员工与权限、物流与顺丰、押金与免押、通知模板、外部接口凭证、系统健康。
- 主操作：保存门店和非敏感配置。
- 次操作：Demo 角色切换、刷新配置 / 健康。
- 缺失表达：凭证只显示已配置 / 未配置，不显示 value。
- Drawer / 详情：模块化面板；高级技术信息只在系统健康里展示。
