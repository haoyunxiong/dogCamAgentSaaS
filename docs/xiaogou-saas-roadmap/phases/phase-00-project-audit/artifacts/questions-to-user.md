# Phase 00 需要用户确认的问题

执行时间：2026-06-12

这些问题是 Phase 00 盘点后无法仅凭代码确定的事项。请后续由用户或业务负责人判断，Codex 不自行猜测。

## 产品范围确认

1. 新产品正式显示名是否固定为“小狗相机助手商户版”，还是前端继续显示“小狗相机助手”？
2. “控制台”是否保留“打开闲鱼 IM”入口，但降级为渠道入口？
3. “会话协同”分组是否在 Phase 01 中整体降级为 legacy 区域？
4. Phase 01 是否只做导航、命名、模块定位，不动业务逻辑？

## 功能保留确认

1. 市场监控是否仍有业务价值？它当前明确追踪闲鱼同类机型帖子。
2. 闲管家开放平台是否仍在实际使用？
3. 消息审批是否未来要支持多渠道客服，还是只作为闲鱼 legacy 保留？
4. 学习审核是否要转为“员工话术/知识沉淀”，还是未来可舍弃？
5. 浏览器只读采集模式是否仍需要保留？

## 数据确认

1. 当前真实生产数据主要在 MySQL `xianyu_agent` 中，还是仍有 SQLite `app_config.db` / `chat_history.db` 必须保留？
2. `data/`、`data/db-backups/`、`data/local/db-backups/` 中哪些是必须保留的生产备份？
3. 是否存在外部 launcher（例如桌面双击脚本）不在当前目录内，需要后续纳入盘点？
4. 是否允许后续阶段读取 MySQL 实际库结构？Phase 00 本次未连接数据库。
5. 后续是否保留 `source_channel = xianyu` 作为历史订单默认值？
6. 是否优先把 `item_id/chat_id/session_id` 抽象为 channel 字段，而不是立刻改旧字段？

## 业务流程确认

1. 订单状态当前哪些是业务真实状态，哪些只是 UI/历史兼容状态？
2. 归还/验机当前是否已有线下 SOP，是否需要在 Phase 05 文档化？
3. 免押平台的真实名称和接口归属是否固定？当前代码中存在免押开放平台/云上印鉴等表达。
4. 顺丰寄件是否是唯一物流渠道，还是未来要支持京东、德邦、手工物流？
5. 店铺维度当前是否只代表内部门店，还是已经接近未来商户/门店模型？

## 迁移策略确认

1. Phase 01 是否以产品定位、信息架构、命名统一为主，不触碰 Electron/Python/DB 核心逻辑？
2. 是否把 `stores` 扩展为 SaaS 底座前的“门店雏形”，还是 Phase 03 重新设计商户/门店模型？
3. `CR-0001_配置中心与敏感配置治理` 是否确认放到 Phase 03，而不是 Phase 00/01？
4. 当前旧 Phase 00 artifacts 是否全部以本次正式仓库中的 artifacts 为准？
5. 用户是否确认 Phase 00 artifacts 通过后，才允许 Codex 准备 Phase 01 计划？
