# Phase 00：当前项目盘点与旧逻辑冻结

## 阶段目标

先不改业务代码，只让 Codex 看懂当前项目，标记旧闲鱼强绑定逻辑，建立后续阶段化执行规则。

## 本阶段 Codex 做什么

- 扫描项目结构
- 输出路由/页面/组件清单
- 输出现有核心能力清单
- 输出 legacy 闲鱼逻辑清单
- 建立 artifacts/analysis-report.md
- 不修改业务逻辑

## 本阶段你做什么

- 确认哪些功能必须保留
- 确认哪些闲鱼功能可以舍弃
- 确认新产品定位
- 确认 Phase 00 盘点结果

## 本阶段交付物

- 当前项目分析报告
- 保留功能清单
- legacy 功能清单
- 风险清单

## 进入下一阶段条件

- 项目结构说明完成
- 保留功能清单完成
- 废弃/legacy 功能清单完成
- AGENTS.md 生效
- 无业务代码改动

## 执行原则

- 先计划，后执行。
- 不跨阶段。
- 不破坏现有可用流程。
- 结果写入本阶段 `artifacts/`。
