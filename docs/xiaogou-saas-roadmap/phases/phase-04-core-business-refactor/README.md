# Phase 04：核心业务流程重构

## 阶段目标

把查档期、创建订单、免押、发货、归还、完结等流程标准化，闲鱼降级为渠道。

## 本阶段 Codex 做什么

- 重构工作台/订单/档期/设备/免押/物流核心页面
- 订单来源 channel 化
- 标准化订单状态和设备状态
- 保留现有有效流程
- 输出真实订单测试报告

## 本阶段你做什么

- 提供真实业务流程
- 确认必填字段
- 确认状态流转
- 用真实订单测试

## 本阶段交付物

- 核心业务页面重构
- 状态流转文档
- 真实订单测试报告

## 进入下一阶段条件

- 创建订单到完结可跑通
- 查档期/免押/发货/归还不断点
- 10 条真实订单测试通过
- 闲鱼不再是核心对象

## 执行原则

- 先计划，后执行。
- 不跨阶段。
- 不破坏现有可用流程。
- 结果写入本阶段 `artifacts/`。
