# 08 Codex 每次执行手册

这份文件解决两个问题：

1. 你每次让 Codex 执行前应该做什么；
2. 你应该怎么问，才能让 Codex 按阶段稳定执行。

---

## 一、每次执行前你要先判断 3 件事

### 1. 当前属于哪个阶段？

看：

```text
docs/xiaogou-saas-roadmap/05_PHASE_STATUS_阶段状态表.md
```

不要跳阶段。

### 2. 这次任务属于哪一类？

只选择一种：

```text
A. 只分析，不改代码
B. 只更新文档
C. 只操作 Figma
D. 只改前端 UI
E. 只改后端/数据库
F. 只做测试验证
G. 只做阶段验收
H. 只做复盘和规则沉淀
```

### 3. 这次是否有风险？

高风险任务包括：

```text
数据库迁移
删除旧逻辑
登录权限
数据隔离
核心订单流程
核心档期流程
生产数据
```

高风险任务必须要求 Codex 输出：

```text
备份方案
回滚方案
影响范围
测试方案
```

## 二、每次固定操作流程

### Step 1：让 Codex 进入计划模式

```text
请先进入计划模式，不要修改代码，不要操作 Figma。
```

### Step 2：限制读取范围

```text
本次只读取 AGENTS.md、总控入口、阶段状态表和当前阶段文件夹，不要读取所有历史 MD。
```

### Step 3：让 Codex 输出计划

必须输出：

```text
当前阶段目标
本次任务目标
本次不做什么
准备读取哪些文件
准备修改哪些文件
风险点
验证方式
需要我确认的问题
```

### Step 4：你确认后再执行

你可以回复：

```text
确认，按这个计划执行。注意不要跨阶段，不要修改数据库，不要删除旧逻辑。
```

### Step 5：执行后要求验收报告

让 Codex 输出：

```text
改了什么
没改什么
涉及文件
如何验证
验证结果
风险点
下一步建议
是否需要更新阶段状态
```

### Step 6：让 Codex 更新记录

如果任务完成，让 Codex 更新：

```text
当前阶段 artifacts/
06_DECISION_LOG_决策记录.md
05_PHASE_STATUS_阶段状态表.md（仅在需要时）
```

## 三、最常用 Prompt 模板

### 1. 启动当前阶段

```text
你现在是“小狗相机助手商户版”项目执行代理。
请先不要修改代码、不要操作 Figma。

请读取：
1. AGENTS.md
2. docs/xiaogou-saas-roadmap/00_README_总控入口.md
3. docs/xiaogou-saas-roadmap/05_PHASE_STATUS_阶段状态表.md
4. docs/xiaogou-saas-roadmap/08_CODEX_DAILY_EXECUTION_GUIDE_每次执行手册.md
5. 当前阶段文件夹下的 README、需求、Codex 任务、验收标准、进入下一阶段条件。

请进入计划模式，只输出：
1. 你理解的当前阶段目标
2. 当前阶段不做什么
3. 本次准备读取哪些项目文件
4. 本次准备产出什么
5. 风险点
6. 需要我确认的问题

在我确认前，不要修改代码。
```

### 2. 执行已确认计划

```text
确认，按刚才计划执行。
执行要求：
1. 不要跨阶段；
2. 不要修改未列入计划的文件；
3. 如果发现必须改计划，先停下来问我；
4. 执行完输出变更清单、验证方式、风险点；
5. 必要结果写入当前阶段 artifacts/。
```

### 3. 阶段验收

```text
请按当前阶段的 06_ACCEPTANCE_CRITERIA_验收标准.md 和 07_NEXT_PHASE_GATE_进入下一阶段条件.md 做阶段验收。
不要改代码。
请输出：
1. 每条验收项是否通过；
2. 未通过原因；
3. 需要补做的任务；
4. 是否建议进入下一阶段；
5. 如果建议进入下一阶段，请给出阶段状态表更新建议，但不要直接更新，等我确认。
```

### 4. 复盘并沉淀规则

```text
请复盘本次任务中出现的问题。
如果发现 Codex 反复犯同类错误，请提出 AGENTS.md 或当前阶段文档的更新建议。
不要直接修改，先给我看建议。
```

## 四、你每次怎么判断 Codex 是否跑偏

出现下面情况就立刻叫停：

```text
开始围绕闲鱼自动回复做架构
没有计划就改代码
一次性改很多无关文件
跳到商户调货/代发闭环
没有备份就改数据库
把手机端做成 PC 缩小版
把 PC 端做成手机放大版
绕过设计系统写新样式
只说完成了但没有验证方式
```

叫停 Prompt：

```text
停止当前执行。
你已经偏离当前阶段范围。
请回到 AGENTS.md 和 05_PHASE_STATUS_阶段状态表.md，重新说明：
1. 当前阶段目标是什么；
2. 你刚才哪里越界了；
3. 接下来如何回到阶段范围内。
不要继续修改代码。
```

## 五、阶段推进节奏

每阶段都按这个节奏：

```text
计划 → 执行 → 验证 → 记录 → 验收 → 进入下一阶段
```

不要：

```text
想到什么做什么
今天改 UI，明天改数据库，后天又改产品定位
```

## 六、给你自己的检查清单

每次找 Codex 前，先问自己：

```text
这次是哪个阶段？
这次只做一件事了吗？
这次有没有可能破坏现有功能？
是否需要备份？
是否需要截图验证？
是否需要更新阶段状态？
这次完成后能不能进入下一步？
```
