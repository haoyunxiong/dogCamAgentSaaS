# 04 Codex 执行纪律

## 每次执行固定步骤

1. 读取 AGENTS.md。
2. 读取总控入口和阶段状态表。
3. 只读取当前阶段文档。
4. 输出计划。
5. 等用户确认。
6. 小步执行。
7. 自测验证。
8. 输出结果和风险。
9. 更新 artifacts 与阶段状态。

## 不允许的行为

- 不经确认直接大改。
- 不经确认修改数据库。
- 不经确认删除 legacy 代码。
- 跨阶段实现未来功能。
- 用新的 UI 风格替换已确认设计系统。
- 只说“完成了”但不说明验证方法。

## 推荐执行粒度

每次只做一种任务：

```text
项目盘点
文档更新
Figma 设计
组件开发
页面重构
数据库迁移
业务流程验证
阶段验收
```

不要一次同时做 Figma、数据库、前端、后端、路由、测试。
