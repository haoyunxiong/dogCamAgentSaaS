# 00 总控入口

## 产品新定位

小狗相机助手商户版不是“闲鱼助手”，而是：

> 面向租赁商家的订单、档期、设备、物流、免押、员工协作和经营数据管理系统。

第一阶段重点不是平台交易，也不是商户之间调货，而是把租赁业务流程标准化、系统化，让新员工也能快速上手。

## 当前阶段路线

| 阶段 | 名称 | 核心目标 |
|---|---|---|
| Phase 00 | 当前项目盘点与旧逻辑冻结 | 看懂项目，标记 legacy，建立执行规则 |
| Phase 01 | 产品定位与系统骨架固定 | 固定产品定位、导航、模块、业务主流程 |
| Phase 02 | 设计系统与 Figma 基础组件 | 建立 UI 设计系统和核心组件 |
| Phase UI-V2 Demo | 新产品体验独立验证 | 暂停旧系统页面改造，独立验证 PC + Mobile 新产品体验 |
| Phase 03 | SaaS 底座 | 登录、商户、门店、员工、权限、数据隔离 |
| Phase 04 | 核心业务流程重构 | 标准化订单、档期、设备、免押、物流流程 |
| Phase 05 | 员工 SOP 与新员工上手系统 | 任务中心、检查清单、话术、操作指引 |
| Phase 06 | 经营数据与报表中心 | 收入、设备 ROI、渠道、效率、异常数据 |
| Phase 07 | 多商户试用版 / 商户 Beta | 外部商户试用、初始化、反馈闭环 |
| Phase 08 | 商户协作网络预留 | 调货、代发、二手交易先规划不闭环开发 |

## 每次执行读取规则

Codex 每次执行只需要读取：

```text
AGENTS.md
00_README_总控入口.md
05_PHASE_STATUS_阶段状态表.md
08_CODEX_DAILY_EXECUTION_GUIDE_每次执行手册.md
当前阶段文件夹下的 README、需求、任务、验收、进入下一阶段条件
```

不要让 Codex 每次读取所有历史文档。

## 当前项目改造原则

```text
先固定结构，再做页面；
先做设计系统，再做 UI；
先用独立 UI-V2 Demo 定风格，再迁移旧系统能力；
先做数据隔离，再给外部商户用；
先做内部提效，再做商户协作；
每次小步执行，每阶段独立验收。
```

## 2026-06-14 路线调整

Phase 02B Figma 设计系统已完成并保留为设计资产。Phase 02C 真实系统 UI 改造暂停在 02C-02，不继续在旧系统页面上做视觉和交互重构。

当前新增 `Phase UI-V2 Demo`：在 `prototypes/ui-v2-demo` 中独立验证未来 PC + Mobile 产品体验。该 Demo 只使用 mock 数据，不接 Electron、不接旧系统业务逻辑、不接真实 API、不接数据库。Demo 风格和页面结构确认后，再规划旧系统业务能力迁移。
