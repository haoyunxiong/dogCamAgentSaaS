# 免押模块验收清单

本文用于最终联调和截图验收，覆盖 mock、本地回调、real 模式与订单联动。

## 1. 设置页

- 页面：`设置 -> 免押开放平台`
- 需要截图：
  - `appId / appSecret / userEid / notify_url / 海报文案 / 联系电话 / 工作时间`
  - 本地回调地址状态区域，显示 `http://127.0.0.1:1032/api/AliNotify/deposit`
  - 点击 `恢复默认` 后的界面
- 预期：
  - 点击保存后提示成功
  - 重新进入页面仍能看到刚保存的值
  - 点击 `恢复默认` 后，本地展示配置和主进程 `DEPOSIT_*` 真实请求配置同时清空
  - `appId / appSecret` 不以明文自动回显到 renderer localStorage

## 2. Mock 创建闭环

- 页面：`订单履约 -> 某条订单 -> 创建免押单`
- 操作：
  - 在 mock 模式下，从订单创建免押单
  - 手动修改租期开始/结束日期
- 需要截图：
  - 创建前表单
  - 创建成功抽屉
  - 管理页新增记录
- 预期：
  - 押金默认来自设备 `purchase_cost`，不是订单押金
  - 修改租期开始/结束日期后，`租期天数` 会自动重算
  - 创建成功后生成免押单号和二维码
  - 创建成功抽屉按钮文案为 `复制邀约`

## 3. Mock 管理页操作

- 页面：`经营管理 -> 免押管理`
- 操作：
  - 切换顶部状态卡筛选
  - 查看详情
  - 复制邀约
  - 保存海报
  - 取消
  - 完结
  - 删除
  - 批量取消 / 批量完结 / 批量删除
- 需要截图：
  - 列表页
  - 详情 Drawer
  - 批量操作确认框
- 预期：
  - 顶部状态卡计数按当前 `关键词 + 是否最新` 范围统计，不会因为点进某个状态筛选就把其他状态计数压成 0
  - `pending_review / approved` 可取消
  - `pending_review / approved` 可完结
  - `cancelled / completed / rejected` 可删除
  - 删除最新单后，旧单会提升为新的 `最新`

## 4. Real 创建

- 页面：`订单履约 -> 创建免押单`
- 操作：
  - 切换 real 模式后创建免押单
- 需要截图：
  - 创建成功结果
  - 若失败，接口错误提示
- 预期：
  - renderer 不直连真实接口
  - 能拿到真实 `depositOrderNo`
  - 能拿到真实 `qrImageUrl`
  - 失败时不 fallback 到 mock

## 5. Real 详情与管理页

- 页面：`免押管理 -> 详情`
- 操作：
  - 不先点详情，直接从列表点击 `复制邀约`
  - 不先点详情，直接从列表点击 `海报`
  - 打开真实免押单详情
  - 点击复制邀约
- 需要截图：
  - 列表中的来源订单列
  - 详情 Drawer 中状态、租期、邀约文本
- 预期：
  - 列表直接 `复制邀约` 时，若列表缺少 `scheme_url`，会自动补详情后再复制
  - 列表直接 `海报` 时，若列表缺少租期/邀约文本，会自动补详情后再生成
  - `scheme_url` 优先作为复制内容
  - `rent_start_time / rent_end_time` 显示正常日期
  - `rentDays` 显示正常天数，不是 `0天`
  - 如果本地缓存缺少来源订单/型号/设备编号，但真实详情 `deposit_farther` 中带有默认补充内容，详情应能恢复这些字段
  - 来源订单缺失时显示 `-`，不重复显示免押单号
  - 详情区标题为 `邀约信息`，字段名为 `邀约文本`
  - real 模式下不会出现 `取消 / 删除 / 批量取消 / 批量删除` 的真实操作入口，这属于接口能力边界，不是页面故障

## 6. 订单完结联动

- 页面：`订单履约`
- 操作：
  - 打开某条订单的右侧详情 Drawer，观察免押状态卡
  - 点击右侧免押状态卡中的 `复制邀约`
  - 点击右侧免押状态卡中的 `保存海报`
  - 对有免押单的订单执行单条完结
  - 对有免押单的订单执行批量完结
- 需要截图：
  - 订单详情 Drawer 中的免押状态卡
  - 订单详情 Drawer 中点击 `复制邀约` / `保存海报` 后的结果
  - 完结前订单状态与免押状态
  - 完结后订单状态与免押状态
- 预期：
  - 管理页或回调把该订单关联免押单状态改掉后，订单详情 Drawer 中的免押状态卡会自动同步，不需要重新选择订单
  - 即使当前没有打开免押管理页，只有订单履约页处于前台，真实回调后右侧免押状态卡也能更新
  - 若右侧卡片上的免押单缺少邀约文本或租期信息，会先补详情，再完成复制或保存海报
  - 同一订单存在多张免押单时，右侧免押状态卡、复制邀约、保存海报、联动完结都应优先落在 `isLatest=true` 或创建时间最新的那张单
  - 订单完结成功后，最新有效免押单同步完结
  - 批量归还/批量完结中若部分订单失败，免押联动只作用于成功归还的订单，不会误完结失败订单的免押单
  - 关联查找同时兼容 `order.id` 和 `order.order_no`
  - 免押完结失败只提示，不回滚订单完结

## 7. 回调与飞书通知

- 场景：
  - `freeze_success`
  - `unfreeze_success`
  - `customer_reminder`
- 需要截图：
  - 本地回调状态区
  - 免押管理页状态变化
  - 飞书通知消息
- 预期：
  - 回调返回 `{"code":"1"}`
  - `freeze_success + deposit_status=0` 仍显示 `已通过`
  - 若详情同步返回终态，则终态覆盖回调初始状态
  - 飞书通知包含免押单号、状态、来源订单、客户信息
  - 同一免押单在同一状态下，不应因为回调后再次打开列表/详情而重复推送飞书提醒
  - 如果当前管理页按旧状态筛选并且已勾选该单，状态变化后该行应从当前筛选结果移除，已选数量不应残留脏数据
