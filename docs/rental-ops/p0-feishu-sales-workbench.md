# P0 飞书成交工作台方案

## 目标

让销售在手机飞书里完成一条租赁线索从“询问档期”到“生成待发货订单”的核心动作，减少切电脑、查表、重复输入和漏记录。

P0 不追求全自动回复闲鱼买家。P0 的核心是把人工销售的操作变快、变准、可沉淀。

## 适用流程

1. 租客咨询型号、租期、收货日期、城市。
2. 销售在飞书发送查档期命令。
3. 机器人返回可用设备、物流计划、租金、免押条件和建议回复。
4. 租客接受后，销售在飞书补客户姓名、电话、地址、租金。
5. 机器人复用最近一次查档期结果，返回确认建单卡片。
6. 销售点确认，系统创建订单、锁定设备、写入档期。
7. 发货日前，飞书推送待发货卡片。
8. 每天固定时间，飞书主动推送今日/明日发货、归还、逾期待完结列表。

## P0 范围

### 必做

- 飞书查档期：自然语言输入，返回档期、可用设备、物流计划。
- 自动报价：根据型号和租期计算总租金。
- 免押条件：根据型号或设备价值区间返回芝麻信用分、成年、实名审核要求。
- 记档期：复用最近一次查档期结果，只补客户和价格即可建单。
- 确认建单：飞书卡片按钮确认后写订单和档期。
- 待发货/待归还：飞书查询今日/明日待发货、待归还、逾期待完结订单。
- 定时提醒：每天 08:00 推今日履约待办，每天 20:00 推明日履约待办。

### 暂缓

- 自动回复闲鱼买家。
- 自动发布小红书/抖音/闲鱼帖子。
- 大规模竞品爬取。
- 自动创建/完结免押单，除非接口已经确认可用。
- 原生 iOS/Android App。

## 飞书命令设计

### 查档期

```text
@机器人 查档期 pocket4 6.1到6.8 发成都武侯
@机器人 查档期 pocket3 6月1号开始 租8天 发杭州西湖
```

返回内容：

```text
大疆POCKET4 2026-06-01 至 2026-06-08
可用 4/6 台：POCKET4-01、POCKET4-02...
物流：2026-05-30 发货，2026-05-31 到货
租金：xxx 元
免押：芝麻分 xxx+，成年，实名审核，港澳通行证不支持
建议回复：...
```

### 记档期

```text
@机器人 记档期 李双双 13800138000 四川省成都市武侯区天府长岛50号 租金300
```

行为：

- 复用同群最近 10 分钟内查档期的型号、租期、物流计划。
- 如果这条消息显式写了型号或日期，则显式输入覆盖缓存。
- 返回确认建单卡片，默认选择第一个可用设备。

### 待发货/待归还

```text
@机器人 今日待发货
@机器人 明日待发货
@机器人 今日待归还
@机器人 明日待归还
@机器人 今日待办
@机器人 明日待办
```

返回内容：

- 订单号、客户、型号、设备编号、预计发货/归还时间、地址摘要。
- 待发货、待归还、逾期归还分区展示。
- 优先级：逾期归还 > 早发外地 > 早发 > 外地 > 普通发货。
- 操作按钮：确认发货、确认归还。

### 定时推送

```text
08:00-08:10 今日履约提醒
20:00-20:10 明日履约提醒
```

规则：

- 推送目标群取最近一次和机器人交互过的飞书群。
- 早上重点提示当天需要发货、归还、逾期待完结。
- 晚上提前提示第二天需要发货、归还、逾期待完结。
- 远距离且需要早发的订单排在前面。
- 免押单自动创建/完结暂未接入外部接口，当前卡片会提醒人工同步完结。

## 数据设计

### 价格规则

优先复用 `rental_pricing`：

- `model_code`
- `min_days`
- `max_days`
- `daily_price`
- `total_price`
- `deposit`
- `remark`
- `active`

P0 需要补充或明确：

- 同一型号不同天数的总价口径。
- `extension_daily_price`：超过已配置租期后的续租日价。
- 活动价/人工改价是否覆盖规则价。
- 飞书报价展示使用规则价，建单时允许人工输入最终租金。

当前交互做法：

- 页面上按“型号整套配置”维护，而不是一条一条建。
- 用户直接录入：
  - 第 1 天总价
  - 第 2 天总价
  - 第 3 天总价
  - ...
  - 续租日价
- 保存时再展开为 `rental_pricing` 多行，继续复用现有查价逻辑。

### 免押规则

建议新增独立规则表，或者先用配置 JSON：

- `model_code` 或 `price_min/price_max`
- `device_value_min`
- `device_value_max`
- `required_zhima_score`
- `adult_required`
- `real_name_required`
- `unsupported_documents`
- `deposit_amount`
- `remark`

P0 可先做配置化，不急着做复杂页面。

### 动态定价后续增强

P0 不实现动态定价，但要保留后续扩展方向：

- 旺季/节假日涨价：如暑期、法定节假日、618、双11 前后。
- 提前预订天数：离租期较远时保持常规价，临近空档期可降价。
- 周末临近补贴：周四/周五后，若周末起租且库存压力大，可触发小幅降价。
- 库存压力：某型号可租台数较多时，可在设定区间内做降价；库存紧张则不降。
- 人工兜底：飞书查档期只给规则价，最终建单仍允许人工覆盖成交价。

建议后续落地为“基础价 + 调价规则栈”：

- 基础价：当前 `rental_pricing`
- 调价规则：时间因子、节假日因子、库存因子、活动因子
- 输出：建议价、最小可谈价、推荐回复文案

### 询单事件

P0 可以先预留，P1 完整做：

- 来源：飞书、闲鱼、小红书、抖音、人人租。
- 型号、租期、城市、是否有档期、报价金额。
- 是否创建订单、未成单原因。

后续规划：

- 每次执行查档期都写一条询单记录。
- 记录字段至少包括：来源、群/会话 ID、操作人、型号、租期、城市、报价、免押门槛、是否有档、可用台数、查询时间。
- 记档期成功后，回写询单记录的 `order_id` 和 `converted_at`。
- 报表按型号输出：询问次数、询问频率、无档率、建单数、转化率、平均报价、平均成交价。
- 这些数据后续用于价格调整、加机器/卖机器判断，以及自动回复策略优化。

## P0 实施顺序

### P0.1 报价和免押规则

目标：飞书查档期能返回租金和免押条件。

任务：
- 确认租金规则来源。
- 确认免押条件规则。
- 实现价格计算函数。
- 实现免押条件解析函数。
- 给查档期卡片增加报价和免押说明。

验收：
- 输入 `查档期 pocket4 6.1到6.8 发成都武侯`，返回租金和免押条件。

### P0.3 记档期确认卡片增强

目标：确认建单前信息足够完整。

任务：
- 卡片展示店铺、型号、租期、设备、租金、客户、地址。
- 默认店铺为小狗相机。
- 支持按钮确认建单。
- 支持多个可用设备按钮，销售可直接选择哪台设备建单。
- 查档期结果附带可直接复制给租客的建议回复。
- 缺字段时返回补充提示。

验收：
- 查档期后只发客户信息，也能生成确认建单卡片。

### P0.4 待发货/待归还查询

目标：手机飞书能查看今日/明日待发货、待归还和逾期待完结。

任务：
- 增加飞书命令解析：今日待发货、明日待发货、今日待归还、明日待归还、今日待办、明日待办。
- 查询订单和物流计划。
- 生成履约待办卡片。
- 支持定时主动推送。

验收：
- 飞书发送 `今日待发货` 返回待发货列表。
- 飞书发送 `今日待办` 返回待发货、待归还、逾期归还分区。

### P0.5 发货/归还处理最小闭环

目标：发货和归还动作能从飞书进入订单记录。

任务：
- 支持标记已发货。
- 支持标记已归还。
- 记录发货时间和物流状态。
- 归还后提示人工同步完结免押单。

验收：
- 飞书卡片可把订单状态推进到已发货。
- 飞书卡片可把订单状态推进到已归还。

### P0.6 手机 H5 最小工作台

目标：手机上可以快速查看履约待办，复杂详情后续扩展。

任务：
- `/mobile` 首页。
- 今日待办。
- 明日待办。
- 今日待发货。
- 今日待归还。

验收：
- 手机浏览器打开 `/mobile`，可查看履约待办。

## 验证清单

- 飞书查档期能走顺丰时效。
- 飞书查档期能返回价格和免押条件。
- 飞书查档期结果能被 10 分钟内的记档期复用。
- 记档期确认后订单写入 `rental_orders`。
- 档期写入 `schedule_blocks`。
- 设备占用在档期页可见。
- 今日待发货能查到刚建的订单。
- 今日待归还能查到当天归还订单。
- 逾期归还能提示需要完结处理的订单。
- 履约待办接口能被手机 H5 调用。
- ngrok/公网回调、飞书事件、MySQL 都正常。
