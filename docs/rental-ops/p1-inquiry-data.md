# P1 询单数据沉淀

## 目标

记录每一次有效查档期，后续能按型号、店铺、来源统计被询问频率、无档率、转化率和成交价格，为定价和增购/出售决策提供数据。

## 已实现范围

- 新增 `inquiry_events` 表。
- 手机 H5 查档期写入询单事件。
- 飞书查档期写入询单事件。
- 查档期响应返回 `inquiryEventId`。
- 飞书“查档期 -> 记档期”上下文会携带 `inquiryEventId`。
- 手机 H5 “使用上次查询建单”会携带 `inquiryEventId`。
- 建单成功后关联询单事件到订单，并记录最终成交价。
- 新增查询接口：
  - `GET /api/mobile/inquiries`
  - `GET /api/mobile/inquiry-stats`
- 桌面助手新增 `询单分析` 页面：
  - 汇总询单次数、有档次数、转化率、平均报价/成交均价。
  - 按型号查看询单排行、无档次数、有档未成单次数、建单转化率。
  - 查看最近询单记录，确认每次查档期是否被记录和是否转化。

## 统计口径

- `inquiries`：有效查档期次数。
- `available`：查档期时存在可用设备的次数。
- `unavailable`：查档期时无可用设备或查询失败的次数。
- `converted`：已关联订单的询单次数。
- `availableNotConverted`：有档期但未建单次数。
- `conversionRate`：`converted / inquiries`。
- `avgQuotedPrice`：有报价记录的平均报价。
- `avgFinalFee`：已成交订单的平均成交价。
